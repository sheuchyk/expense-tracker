# PR Skill

Создай Pull Request на GitHub для текущей ветки.

## Аргументы

Скилл принимает два аргумента (в произвольном порядке в тексте запроса пользователя):

1. **Название PR** — заголовок Pull Request (на русском языке, по правилам ниже).
2. **Целевая ветка** (base) — ветка, в которую вливается текущая. По умолчанию `master`.

Если пользователь передал только один аргумент — считай его названием PR, а base определи по умолчанию.

## Требования к названию PR

- На **русском языке**
- Соответствует формату Conventional Commits: `type(scope): описание`
- С **маленькой буквы** после двоеточия
- **Без точки** в конце
- Кратко и по существу (до 70 символов)

Допустимые `type` и `scope` — те же, что в скилле [`commit`](../commit/SKILL.md).

## Алгоритм выполнения

1. Определи текущую ветку:
   ```bash
   git rev-parse --abbrev-ref HEAD
   ```
   Если это `master` или `main` — остановись и сообщи пользователю: PR из основной ветки не создаётся.

2. Проверь состояние рабочего дерева:
   ```bash
   git status
   ```
   Если есть незакоммиченные изменения — сообщи пользователю и предложи сначала закоммитить (через скилл `commit`).

3. Посмотри коммиты, которые попадут в PR относительно base-ветки:
   ```bash
   git log <base>..HEAD --pretty=format:'%h %s'
   git diff <base>...HEAD --stat
   ```

4. Убедись, что текущая ветка запушена на remote:
   ```bash
   git push -u origin <current-branch>
   ```

5. Сформируй тело PR по шаблону ниже (на основе коммитов и диффа).

6. Создай PR через `gh` (HEREDOC для тела):
   ```bash
   gh pr create --base <base> --head <current-branch> \
     --title "<название PR>" \
     --body "$(cat <<'EOF'
   ## Описание

   <1–3 предложения о сути изменений>

   ## Изменения

   - <ключевое изменение 1>
   - <ключевое изменение 2>

   ## Как проверить

   - [ ] <шаг проверки 1>
   - [ ] <шаг проверки 2>
   EOF
   )"
   ```

7. Верни пользователю URL созданного PR (`gh pr view --web` не запускать — только показать ссылку из вывода `gh pr create`).

## Важные ограничения

- **Не создавай** PR из `master` / `main`.
- **Не используй** `--no-verify` при пуше.
- **Не пушь** с флагом `--force` без явной просьбы пользователя.
- Если PR для текущей ветки уже существует — сообщи об этом (через `gh pr view`) и не создавай дубликат.
- Название PR **не выдумывай** — используй то, что передал пользователь. Если оно не соответствует правилам (например, на английском или с большой буквы) — предупреди пользователя, но создавай с указанным им заголовком.
