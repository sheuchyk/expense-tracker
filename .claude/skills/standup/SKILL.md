# Standup Skill

Собери детальный отчёт о работе за вчерашний день на основе git-коммитов — для устного отчёта на стендапе.

## Алгоритм выполнения

1. Определи вчерашнюю дату:
   ```bash
   date -v-1d +%Y-%m-%d       # macOS
   ```
   Границы периода: `YYYY-MM-DD 00:00` — `YYYY-MM-DD 23:59`.

2. Узнай текущего автора коммитов:
   ```bash
   git config user.email
   ```

3. Получи список коммитов автора за вчера по всем веткам:
   ```bash
   git log --all --author="$(git config user.email)" \
     --since="YYYY-MM-DD 00:00" --until="YYYY-MM-DD 23:59" \
     --pretty=format:'%h %s (%an, %ad)' --date=short
   ```

4. Для каждого коммита посмотри детали через `git show --stat <hash>` (и, если нужно, `git show <hash>` для содержания дифа), чтобы понять суть изменений — какие файлы, модули, часть стека затронуты.

5. Сгруппируй изменения логически:
   - по scope (backend / frontend / prisma / docs / ci и т.п.)
   - по типу работы (новая функциональность, багфиксы, рефакторинг, документация)

## Формат отчёта

Отчёт на **русском языке**, компактный, но детальный. Структура:

```
📅 Стендап за YYYY-MM-DD (вчера)

✅ Что сделано:

Backend:
- <краткое описание в 1 строке> (`<hash>`)
  - <детали: какие модули/файлы/логика затронуты>

Frontend:
- ...

Прочее (docs, chore, ci):
- ...

📊 Итого: N коммитов, M файлов изменено
```

## Важные ограничения

- Если за вчера коммитов **нет** — честно сообщи об этом и не выдумывай активность.
- Смотри только коммиты **текущего автора** (по `git config user.email`), если пользователь не попросил иное.
- Не пиши WHAT-описания в стиле «изменил файл X» — важна **суть** работы (какая фича, какой баг, зачем).
- Не выполняй никаких изменений в репозитории — скилл только для чтения.
