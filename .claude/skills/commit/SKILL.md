# Commit Skill

Создай git-коммит для текущих изменений, соблюдая соглашения проекта.

## Правила коммитов

Используем [Conventional Commits](https://www.conventionalcommits.org/):

```
type(scope): краткое описание на русском языке
```

### Допустимые type:

- `feat` — новая функциональность
- `fix` — исправление бага
- `refactor` — рефакторинг без изменения поведения
- `docs` — изменения документации
- `chore` — вспомогательные задачи (обновление зависимостей, конфиги)
- `style` — форматирование, пробелы (без логических изменений)
- `test` — добавление или изменение тестов
- `perf` — улучшение производительности
- `ci` — изменения CI/CD
- `build` — изменения системы сборки

### Scope:

- `backend` или `frontend` — для изменений в соответствующей части
- Конкретный модуль: `auth`, `transactions`, `categories`, `prisma`, `users` и т.д.
- Опускай scope, если изменения глобальные (например, `docs:`)

### Требования к описанию:

- На **русском языке**
- С **маленькой буквы**
- **Без точки** в конце
- Кратко и по существу

### Примеры:

```
feat(backend): добавить модуль транзакций
fix(frontend): исправить валидацию формы регистрации
refactor(auth): вынести JWT-логику в отдельный сервис
docs: обновить CLAUDE.md
chore(prisma): добавить индексы на таблицу transactions
perf(frontend): оптимизировать ре-рендер списка транзакций
```

## Алгоритм выполнения

1. Запусти `git status` — посмотри, что изменено
2. Запусти `git diff` (staged + unstaged) — разберись в сути изменений
3. Посмотри последние коммиты (`git log --oneline -5`) для контекста
4. Определи подходящие `type` и `scope`
5. Добавь в staging нужные файлы (`git add <файлы>`) — **не используй `git add -A`**, чтобы не захватить лишнее
6. Сформируй сообщение коммита по правилам выше
7. Создай коммит через HEREDOC:

```bash
git commit -m "$(cat <<'EOF'
type(scope): описание на русском

Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>
EOF
)"
```

8. Проверь результат через `git status`

## Важные ограничения

- **Никогда не коммить** `.env`, `.env.local`, файлы с секретами
- **Не пушь** автоматически — только коммит, если пользователь не попросил иное
- **Не используй** `--no-verify`, `--amend` без явной просьбы пользователя
- **Не добавляй** файлы без явного понимания их содержимого
